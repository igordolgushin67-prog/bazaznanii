# ПРИНЦИПЫ РАБОТЫ С ИНСТРУКЦИЕЙ ДЛЯ АССИСТЕНТОВ

## 1. Базовая философия

### 1.1 Инструкция живёт в репозитории
Главная инструкция (версия X.Y) всегда находится в папке `versions/`. Она не копируется в "ядро" ассистента, не зашивается в system prompt навсегда и не хранится в памяти платформы.

### 1.2 Ссылка как единственный источник
Каждый новый диалог начинается со ссылки на актуальную версию инструкции. Ассистент обязан прочитать её и использовать как ЕДИНСТВЕННУЮ ОСНОВУ для всего диалога.

### 1.3 Итеративность
Опыт, накопленный в диалоге, фиксируется в новых файлах в `archive/опыт/`. Этот опыт может быть использован для создания следующей версии инструкции.

## 2. Как это работает на разных платформах

### 2.1 Общий принцип
Инструкция передаётся через механизм, который платформа предоставляет для задания поведения ассистента.

### 2.2 ChatGPT (веб-версия)
Достаточно вставить ссылку на инструкцию в первом сообщении. Ассистент прочитает её и будет следовать правилам.

### 2.3 OpenAI API / Azure OpenAI
Инструкция передаётся в параметре `system_prompt` при создании диалога.

### 2.4 Microsoft Semantic Kernel
Инструкция указывается в поле `Instructions` при создании агента через `AgentDefinition`.

### 2.5 IBM watsonx
Инструкция вставляется в поле `Instructions` в настройках поведения агента.

### 2.6 Любая другая платформа
Найти поле, которое задаёт "системные инструкции" или "поведение", и вставить туда ссылку на файл.

## 3. Критически важные правила

### 3.1 Абсолютный приоритет инструкции
Правила из инструкции имеют приоритет над любыми общими знаниями ассистента о SUNO AI.

### 3.2 Приоритет пользовательских правок
Если пользователь даёт ссылку на файл с правками или уточнениями — эти правки имеют приоритет над текущей инструкцией.

### 3.3 Обязательное чтение
При получении любой ссылки на файл репозитория ассистент ОБЯЗАН перейти по ней и прочитать содержимое.

### 3.4 Навигация по проекту
Файл `Suno/STRUCTURE.md` содержит полную карту всех папок и файлов. Его нужно использовать для точного понимания структуры.

## 4. Почему это лучше, чем загрузка в "ядро"

| Критерий | Загрузка в ядро | Ссылка в начале диалога |
|----------|-----------------|------------------------|
| Актуальность версий | Требует перезагрузки ядра | Всегда последняя версия |
| Несколько диалогов | Одна версия для всех | Можно тестировать разные |
| Итеративность | Сложно обновлять | Опыт → новый файл → новая версия |
| Техническая сложность | Высокая | Минимальная |
| Прозрачность | Чёрный ящик | Полная видимость правил |
